## Fix 3: #37 — Guard-System False Positive + Pattern Cleanup

### 3a. Centralize shared patterns

Create `plugin/scripts/infrastructure/patterns.sh`:

```bash
# Shared path patterns for hook enforcement.
[ -n "${_WFM_PATTERNS_LOADED:-}" ] && return 0
_WFM_PATTERNS_LOADED=1

# Enforcement files — protected in ALL phases including implement/review
GUARD_SYSTEM_PATTERN='(\.claude/hooks/|(^|[^a-z-])plugin/scripts/|(^|[^a-z-])plugin/commands/)'

# State files — protected from direct writes in ALL phases
STATE_FILE_PATTERN='\.claude/(state/workflow\.json|state/phase-intent\.json)'
```

### 3b. Fix guard-system check in pre-tool-bash-guard.sh

Current (broken — matches full command string):
```bash
GUARD_SYSTEM_PATTERN='(...)'
if echo "$COMMAND" | grep -qE "$GUARD_SYSTEM_PATTERN"; then
    if _detect_write_operation "$COMMAND"; then
        emit_deny ...
```

Fixed (matches extracted write target only):
```bash
source "$SCRIPT_DIR/infrastructure/patterns.sh"
# ...
if _detect_write_operation "$COMMAND"; then
    local write_target
    write_target=$(_extract_write_target "$COMMAND")
    if [ -n "$write_target" ] && echo "$write_target" | grep -qE "$GUARD_SYSTEM_PATTERN"; then
        _log "DENY: write to enforcement file ($write_target)"
        emit_deny "$(_phase_deny_message "$PHASE" "guard-system")"
        exit 0
    fi
fi
```

Same fix for STATE_FILE_PATTERN check (lines 70-77).

### 3c. Fix guard-system check in pre-tool-write-gate.sh

Write-gate already uses `$NORMALIZED_PATH` (line 60), not the full command. But the pattern is hardcoded inline. Replace with sourced constant from `patterns.sh`.

### 3d. Remove duplicate redirect stripping

`pre-tool-bash-guard.sh:60` has an inline sed that duplicates `write-patterns.sh:23`. Remove it — `_detect_write_operation()` already handles redirect stripping internally.

### 3e. Fix _gh_safe_chain to allow || gh fallback

Current `_gh_safe_chain` strips `|| true` and `|| echo ...` but rejects `|| gh ...`. Fix the sed to also strip `|| gh ...` suffixes:

```bash
stripped=$(echo "$cmd" | sed -E 's/\|\|[[:space:]]*(true|echo[[:space:]][^;&|]*|gh[[:space:]].*)$//')
```

This allows `gh issue list ... || gh issue list ...` fallback patterns.

### 3f. Add gh api to allowed read-only commands

The `_check_gh_command` function (gh-safety.sh:25) allows `issue view`, `issue list`, `pr view`, `pr list`, etc. but doesn't allow `gh api` for read-only API calls. Add `api` to the allowlist for define/discuss phases:

```bash
gh[[:space:]]+(repo[[:space:]]+view|issue[[:space:]]+(view|list|comment)|pr[[:space:]]+(view|list)|release[[:space:]]+(view|list)|api[[:space:]])
```

## Fix 4: #35 — In-Place State Updates

### 4a. Replace jq -n rebuild with _update_state

Both `user-set-phase.sh` and `agent-set-phase.sh` rebuild state from scratch. Replace with in-place updates.

```bash

    '.phase = $p | .message_shown = false | .active_skill = "" | .coaching = {tool_calls_since_agent: 0, layer2_fired: []}' \
    --arg p "$new_phase"
```

**Off transition (clears cycle fields):**
